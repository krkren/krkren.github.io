BinaryStream.copy

Function/Meaning
Copy data from another storage
Type
Method of the BinaryStream class
Syntax
copy(storage, elm)
Arguments
storage Source storage (An exception occurs if it is the same as the currently open storage)
elm Dictionary for operation specification (in/out). Contents are as follows: (When omitted, a simple copy is performed, no [out] parameters)
offset :[in] Read start offset of the source storage (defaults to the beginning if omitted)
length :[in] Maximum read size of the source storage (defaults to file size if omitted)

filter :[in] Function name to perform filtering (no filtering if omitted)
fparam :[in] Parameter value to pass to the filter (int)

nocopy :[in] If true is specified, output will not be written (defaults to false; specify this if you only want to obtain the hash, etc.)
md5 :[in] If true is specified, returns the MD5 digest of the output

hash :[out] Hash value of the read file (Adler32)
read :[out] Number of bytes read
digest :[out] MD5 digest of the written data (octet) * Only valid if md5 is true
Return Value
Number of bytes written
Description
The contents of the target storage from the offset for the length are written starting from the current stream position.
Refer to the description of setFilter() for details on the filter.
