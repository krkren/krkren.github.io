Window.bringTo

Function/Meaning
Changes the Z-order of the window
Type
Method of the Window class
Syntax
bringTo(win, activate)
Arguments
win Window object, specified string, or numeric value
If it's a Window object, this window is placed behind that window
If it's a string, one of "bottom", "notopmost", "top", or "topmost" (equivalent to HWND_*) cf. SetWindowPos API
If it's a number, it is treated as a window handle or a direct specification of an HWND_* macro value
#define HWND_TOP ((HWND)0)
#define HWND_BOTTOM ((HWND)1)
#define HWND_TOPMOST ((HWND)-1)
#define HWND_NOTOPMOST ((HWND)-2)
activate Flag for whether to activate the window
Even if set to true, the application itself does not become active (no taskbar flashing, etc., will occur)
Return Value
None (void)
Description
