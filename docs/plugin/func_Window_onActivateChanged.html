Window.onActivateChanged

Function/Meaning
■Extended Event: Active State Change Notification
Type
Window class method
Syntax
onActivateChanged(active, minimize)
Arguments
active 0: Deactivated, 1: Activated by non-mouse click, 2: Activated by mouse click
minimize 0: Window is not minimized, Other: Minimized
Return Value
None (void)
Description
* The difference from Window.onActivate and Window.onDeactivate is (as applies to all extended events) that the event callback is triggered even if System.eventDisabled == true.

For example, by setting System.eventDisabled = true during System.onDeactivate (which would prevent System.onActivate from being called) and restoring it using this event notification, you might be able to implement features like "pause operations when inactive" (however, since this only stops event delivery, it may not be very meaningful for applications using timers that calculate the elapsed time from the previous frame).
Note that when System.eventDisabled = true, even screen updates do not run, so there is an issue where the screen may become corrupted if WM_PAINT occurs, such as when another window is placed on top (in such cases, use OverlayBitmap, etc., described later).
