Layer.shrinkCopy

Function/Meaning
Shrink copy
Type
Method of Layer class
Syntax
shrinkCopy(dleft, dtop, dwidth, dheight, src, sleft, stop, swidth, sheight)
Arguments
dleft Left edge of the destination rectangle. Decimals allowed.
dtop Top edge of the destination rectangle. Decimals allowed.
dwidth Width of the destination rectangle. Decimals allowed.
dheight Height of the destination rectangle. Decimals allowed.
src Source layer
sleft Left edge of the source rectangle. In pixels. Decimals not allowed.
stop Top edge of the source rectangle. In pixels. Decimals not allowed.
swidth Width of the source rectangle. In pixels. Decimals not allowed.
sheight Height of the source rectangle. In pixels. Decimals not allowed.
Return Value
None (void)
Description
An error occurs if the copy is enlarged (dwidth > swidth || dheight > sheight).
The reduction algorithm used is the area averaging method. It provides higher quality than affineCopy for reductions of 1/2 or smaller.
Unlike stretchCopy, Layer.face and holdAlpha are ignored.

* Note: False colors (colors hidden behind completely transparent parts) may appear at the boundaries between transparent and opaque areas, so it is recommended to use Layer.oozeColor() from layerExSave.dll or similar methods beforehand.
