Scripts.rehash

Function/Meaning
Calls TJSDoRehash()
Type
Method of the Scripts class
Syntax
rehash()
Arguments
Return Value
None (void)
Description
Sets the hash table size reconstruction flag for objects based on TJSCustomObject.
(This applies to Dictionaries and most class and instance objects.)

Explanation:
The hash table size is determined by the number of elements held by that object.
After the flag is set, a recalculation is performed the next time something is written to the object,
and reconstruction occurs only if the table size needs to change. (Size ≈ 2^(bits of element count + 2)?)
Normally, Kirikiri calls TJSDoRehash() periodically (during idle time and when at least 1.5 seconds have passed since the last call), so there is usually no need to worry about it.
However, when adding a large number of elements without returning to an idle state,
it may be effective to call this at intervals, such as once every several thousand or ten thousand additions using a counter.
