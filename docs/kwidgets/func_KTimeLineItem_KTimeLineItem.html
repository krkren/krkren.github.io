KTimeLineItem.KTimeLineItem

Function/Meaning
Constructor
Type
Method of KTimeLineItem class
Syntax
KTimeLineItem(label, name)
Arguments
label Label
name Name

Creates a timeline item.

KTimeLineItem is a class that extends KListItem and,
in addition to all behaviors of a list item, has the functionality to
manage frame information for each timeline.

A frame is defined as an object with four fields: time, type, and content.
time is the start time of the frame,
and content is user-defined frame information.
type is the frame type. It will be one of the following four types:
- TIMELINE_FRAME_TYPE_NULL: Null frame. Treated as an area where no frame exists.
- TIMELINE_FRAME_TYPE_SINGLE: Single frame. A non-continuous frame that occupies only one unit.
- TIMELINE_FRAME_TYPE_CONTINUOUS: Continuous frame. A frame that occupies a continuous area until the next frame.
- TIMELINE_FRAME_TYPE_TWEEN: Tweening frame. A frame that connects continuous areas while interpolating until the next frame.

Use the following functions to refer to existing frames.
You can overwrite frame information by directly rewriting the content of the referenced frame, but
since frame objects are frequently updated by frame operations,
it is not recommended to continue referencing a frame for a long time.
You should reference it every time you edit.

- frameCount: Refers to the number of frames
- frameAt(index): Refers to a frame
- frameLengthAt(index): Refers to the length of a frame
- findFrame(time): Gets the index of the frame corresponding to the time.
- lastTime: Gets the time of the last frame
- getCompleteFrameContent(time): Calculates frame information corresponding to the time

Use the following frame operation functions to manipulate frames.

- insertKeyFrame(time, type): Inserts a keyframe.
- deleteKeyFrame(time): Deletes a keyframe.
- splitFrame(time): Splits a frame.
- deleteAllFrame(): Deletes all frames.

Use the following functions to copy and paste frames within a specific range:

- copyFrameRange(beginTime, endTime): Copies frames
- cutFrameRange(beginTime, endTime): Cuts frames
- pasteFrame(time, frameList): Pastes frames

When a frame operation function is called, the following functions are called
internally as needed. Users must implement these functions appropriately.

- tweenFrameContent(fromContent, toContent, ratio): Performs interpolation between frames.

The following interpolation functions are provided to perform frame interpolation calculations.

- tweenIntValue(fromValue, toValue, ratio): Linearly interpolates integers

Additionally, the following properties exist to set the appearance and behavior of frames.

- singleFrameColor: Sets the background color of single frames
- continuousFrameColor: Sets the background color of continuous frames
- tweenFrameColor: Sets the background color of tween frames

Use the following functions when redrawing of frames is required due to reasons such as changing frame information.

- redrawFrameRange(fromTime, toTime): Redraws frames in the specified range.
- redrawAllFrame(): Redraws all frames.

By default, it has the following serialization functions.

- serialize(): Serializes the contents of the item into an object.
- unserialize(data): Unserializes the contents of an object.

Also, the following event functions are called at specific timings:

- onTimeLineRightClick(time): Called when the timeline is right-clicked.
- onTimeLineKeyDown(key, shift): Called when a key is pressed on the timeline.
- onFrameDragged(): A frame was dragged.
Return Value
None (void)
